# Contribution Guidelines

Thank you for considering contributing to the Metadata Forensic Tool! We appreciate your help and want to make the contribution process as smooth as possible. Below are the guidelines for contributing to this project.

## Code Style
- Follow the project's coding standards. If you're unsure, look at the existing code for reference.
- Use 2 spaces for indentation.
- Use single quotes for strings instead of double quotes, unless you need to use double quotes inside a string.

## Commit Messages
- Keep the subject line under 50 characters.
- Use the imperative mood for the subject line (e.g., "Add feature" vs. "Added feature").
- Include a detailed description if the change is not self-explanatory. Write the description in the body.

## Pull Request Process
1. **Fork the repository** to your own GitHub account and clone it to your local machine.
2. **Create a new branch** for your feature or bug fix.
3. **Make your changes** and commit them with clear commit messages.
4. **Push your branch** back to your fork on GitHub.
5. **Open a pull request** on the original repository's main branch.
6. **Link any relevant issues** in your pull request description.

## Development Workflow
- Ensure you have all dependencies installed by running `npm install`.
- You can test your changes locally by running `npm test`.
- Make sure all tests are passing before submitting your pull request.
- Review your changes and ensure they follow the style guide before pushing.

If you have any questions or need further assistance, feel free to reach out! Happy coding!
